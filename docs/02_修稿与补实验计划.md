# 修稿与补实验计划（1周可执行）

## 目标

在最短周期内把稿件从“工程可行”提升到“审稿可过”的版本，重点修复一致性、严谨性和可复现性问题。

---

## P0（必须完成）

1. 全文一致性修复  
- 统一数据集描述、类别数、模型规模、是否使用 BiFPN、最终方法名。  
- 摘要、方法、实验、结论、README 口径一致。  
- 清除全部 `XX.X` 占位符。

2. 文献与可信度修复  
- 替换 placeholder 风格参考文献为可验证来源。  
- 检查表格/图注/文中引用一一对应。

3. 实验统计补强  
- 主结果与关键消融至少 3 seeds。  
- 报告 `mean ± std`。  
- 保留原始日志和每次随机种子。

4. 公平对比协议  
- 固定训练预算：输入尺寸、epoch、增强、优化器、评估 split。  
- 若对比方法无法完全一致，明确说明偏差与影响。

---

## P1（强烈建议）

1. 泛化验证  
- 增加一个额外公开基准，或至少跨划分鲁棒性验证。  
- 补充小目标专属分析（面积分桶或 AP_small）。

2. 效率报告  
- Params / FLOPs / FPS / Latency / Model size 全量报告。  
- 统一推理环境（batch=1、同硬件、同预处理）。

3. 误差分析  
- 漏检/误检类型统计。  
- 增加典型可视化案例并解释失败模式。

---

## P2（加分项）

1. 超参敏感性  
- \(C_0\)、\(k\)、\(\alpha\) 的网格或区间稳定性。  
- 给出推荐区间而非单点最优。

2. 训练稳定性  
- 单阶段 vs warm-up 曲线对比。  
- 解释前期波动来源与收敛行为。

3. 理论-实证闭环  
- 画出梯度幅值随尺度变化（IoU vs SA-NWD）。  
- 对应理论结论做实证验证。

---

## 一周排期（Day 1~7）

### Day 1：一致性修稿
- 统一摘要/方法/实验设置口径。
- 修正冲突符号与公式叙事。
- 清理占位符数字和文本。

### Day 2：主结果多种子
- 跑 Baseline 与 Ours 的 3 seeds。
- 整理 mean ± std。

### Day 3：关键消融多种子
- 至少跑 +P2、+SA-NWD、Full 三组各 3 seeds。
- 汇总增益及方差。

### Day 4：效率评测
- 统一条件下测 Params/FLOPs/FPS/Latency/Size。
- 产出效率表和脚注说明。

### Day 5：泛化与误差分析
- 补额外数据验证（或跨划分）。
- 输出失败案例与类型统计。

### Day 6：写作与图表定稿
- 重写结果分析和讨论。
- 完成图表编号、引用、图注一致性检查。

### Day 7：投稿材料打包
- 完成 Response 文档。
- 自检可复现性（命令、配置、随机种子、日志路径）。
- 输出最终投稿包。
