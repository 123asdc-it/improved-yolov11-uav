# 无人机小目标检测项目 — 操作指南

---

## 一、技术背景

### 1.1 YOLOv11n 基线介绍

YOLOv11 是 Ultralytics 于 2024 年发布的最新 YOLO 系列检测器。YOLOv11n（nano）是最轻量的版本：

| 指标 | 数值 |
|------|------|
| 参数量 | 2.6M |
| 计算量 | 6.6 GFLOPs |
| 层数 | 181 |
| 检测头 | 3 个（P3/P4/P5，stride 8/16/32） |

**核心架构：**

```
输入图像
  │
  ├── Backbone（特征提取）
  │     Conv → Conv → C3k2 → Conv → C3k2 → Conv → C3k2 → Conv → C3k2 → SPPF → C2PSA
  │     P1/2   P2/4          P3/8          P4/16         P5/32
  │
  ├── Neck（特征融合，PANet 双向路径）
  │     上采样 + Concat → C3k2（自顶向下融合大尺度语义信息到小尺度）
  │     下采样 + Concat → C3k2（自底向上融合小尺度细节到大尺度）
  │
  └── Head（检测头）
        Detect(P3, P4, P5) → 分别负责小/中/大目标
```

**关键模块：**
- **C3k2**：CSP Bottleneck 的变体，使用两个卷积核交叉连接，提取多尺度特征
- **C2PSA**：Cross Stage Partial + Spatial Attention，在特征提取末端增强空间注意力
- **SPPF**：Spatial Pyramid Pooling - Fast，多尺度池化聚合全局上下文信息

**YOLOv11n 在本任务的局限**：默认最小检测 stride=8，对于无人机图像中仅 ~50×50 像素的极小目标，特征图分辨率不够。

---

### 1.2 四项改进详解

#### 改进 1：P2 小目标检测头

**问题**：默认 3 个检测头（stride 8/16/32）对小目标检出率不足。

**方案**：新增 stride=4 的 P2 检测头，利用更高分辨率特征图。

```
原始：Detect(P3, P4, P5)    → 3 个检测头
改进：Detect(P2, P3, P4, P5) → 4 个检测头
                 ↑
          stride=4，特征图为输入的 1/4
          1280 输入 → 320×320 的检测网格
```

**原理**：P2 层保留了更多空间细节信息。对于 1280×1280 输入，P2 层生成 320×320 的特征图（比 P3 的 160×160 大 4 倍），每个网格单元覆盖原图仅 4×4 像素区域，大幅提升小目标的定位精度和检出率。

**论文来源**：TPH-YOLOv5（ICCV Workshop 2021）首次在 YOLO 中使用高分辨率检测头用于无人机场景。

---

#### 改进 2：EMA 注意力机制

**问题**：小目标容易淹没在复杂背景中，需要增强特征表达能力。

**方案**：在 backbone 的 P3 和 P4 层后插入 EMA（Efficient Multi-Scale Attention）。

```
原始：C3k2 → Conv → ...
改进：C3k2 → EMA → Conv → ...
```

**原理（3 步）**：
1. **分组 + 空间池化**：将通道分成 G 组，分别沿 H 和 W 方向做自适应平均池化，捕获空间位置信息
2. **双路径交互**：一路 1×1 卷积、一路 3×3 卷积，分别提取全局和局部特征，然后通过矩阵乘法做交叉注意力
3. **权重调制**：用 sigmoid 激活的注意力权重调制原始特征

**优势**：参数增量 < 0.1M，计算开销极低，但能有效抑制背景噪声、聚焦目标区域。

**论文来源**：Ouyang et al., ICASSP 2023

---

#### 改进 3：PConv 轻量化卷积

**问题**：P2 检测头引入额外计算量，需要补偿以保持轻量化。

**方案**：在 Neck 中用 PConv_C3k2 替换标准 C3k2。

```
标准卷积：对全部 C 个通道做 3×3 卷积 → FLOPs = k²·C_in·C_out·H·W
PConv：  只对 C/4 个通道做卷积，其余直接传递 → FLOPs 降低约 75%

┌──────────────────┐
│  输入: C 个通道   │
├─────┬────────────┤
│ C/4 │  3C/4      │
│  ↓  │   ↓        │
│ 3×3 │ Identity   │
│ Conv │ (直接传递) │
├─────┴────────────┤
│ Concat → BN → SiLU│
└──────────────────┘
```

**原理**：相邻通道的特征往往高度冗余。PConv 只处理 1/4 的通道就能提取足够的特征信息，其余通道保持不变，大幅降低计算量而几乎不影响精度。

**效果**：模型参数减少 15-25%，FPS 提升，精度基本持平。

**论文来源**：Chen et al., "Run, Don't Walk: Chasing Higher FLOPS for Faster Neural Networks", CVPR 2023

---

#### 改进 4：BiFPN 加权特征融合

**问题**：标准 Concat 对所有特征图一视同仁，无法区分不同尺度特征的重要性。

**方案**：用 BiFPN 加权 Concat 替换标准 Concat。

```
标准 Concat：Output = Concat(F1, F2)
BiFPN：     Output = Concat(w1/(w1+w2+ε)·F1, w2/(w1+w2+ε)·F2)
                              ↑ 可学习权重（ReLU 约束非负）
```

**原理**：每个输入特征图有一个可学习的标量权重 w_i，通过 Fast Normalized Fusion 归一化后加权。网络在训练中自动学习哪个尺度更重要，对特征融合质量有提升。

**效果**：mAP 提升 1-2%，参数增量可忽略（每个融合点仅增加 2 个标量参数）。

**论文来源**：Tan et al., "EfficientDet: Scalable and Efficient Object Detection", CVPR 2020

---

### 1.3 改进的整体效果

| 组件 | mAP50 影响 | 参数量影响 | FPS 影响 |
|------|:--------:|:--------:|:------:|
| P2 检测头 | ↑↑↑ | 略增 | 略降 |
| EMA 注意力 | ↑ | ≈ 不变 | ≈ 不变 |
| PConv 轻量化 | ≈ 不变 | ↓↓ | ↑ |
| BiFPN 融合 | ↑ | ≈ 不变 | ≈ 不变 |
| **总体** | **显著提升** | **基本持平** | **基本持平** |

核心思路：**P2 大幅提升精度，PConv 补偿计算开销，EMA 和 BiFPN 锦上添花。**

## 二、环境配置（恒源云 RTX 3060）

### 2.1 租用服务器

1. 注册 [恒源云](https://gpushare.com/)
2. 选择 **单卡 RTX 3060**（12GB 显存），推荐 PyTorch 2.0+ 镜像
3. 启动实例后获取 SSH 连接信息

### 2.2 连接服务器

**Xshell 连接：**
```
主机：<恒源云提供的 IP>
端口：<恒源云提供的端口>
用户名/密码：<恒源云提供>
```

**VSCode 远程开发：**
1. 安装 Remote-SSH 插件
2. 按 `Ctrl+Shift+P` → `Remote-SSH: Connect to Host`
3. 输入 `ssh -p <端口> root@<IP>`

### 2.3 安装环境

```bash
# 安装 ultralytics（包含 YOLOv11）
pip install ultralytics

# 安装 ONNX（后续导出模型用）
pip install onnx onnxruntime
```

### 2.4 上传文件

使用 Xftp 或 scp 上传以下文件到服务器：

```bash
# 在服务器上创建项目目录
mkdir -p /root/drone_detection

# 整个项目目录上传（保持目录结构）：
# configs/                     → /root/drone_detection/configs/
# scripts/                     → /root/drone_detection/scripts/
# ultralytics_modules/         → /root/drone_detection/ultralytics_modules/
# datasets/                    → /root/drone_detection/datasets/
```

### 2.5 修改数据集路径

上传后编辑 `configs/data.yaml`，将 `path` 改为服务器上的绝对路径：

```yaml
path: /root/drone_detection/datasets
train: images/train
val: images/val
test: images/test
nc: 1
names: ['drone']
```

## 七、常见问题

**Q: 训练时 GPU 显存不够怎么办？**
A: 将 batch 从 8 降到 4，或将 imgsz 从 1280 降到 960。

**Q: 训练 loss 不收敛？**
A: 检查 data.yaml 的 path 是否正确。小数据集建议关闭 mixup（设为 0）。

**Q: 消融实验某组报错？**
A: 大概率是自定义模块注册问题，确保 `import register_modules` 在最前面。

**Q: 如何训练 SOTA 对比模型？**
A: 用标准 ultralytics 命令即可：
```bash
yolo detect train model=yolov8n.pt  data=data.yaml imgsz=1280 epochs=300 batch=8
yolo detect train model=yolov10n.pt data=data.yaml imgsz=1280 epochs=300 batch=8
```
