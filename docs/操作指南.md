# 无人机小目标检测项目 — 操作指南

---

## 一、技术背景

### 1.1 YOLOv11n 基线介绍

YOLOv11 是 Ultralytics 于 2024 年发布的最新 YOLO 系列检测器。YOLOv11n（nano）是最轻量的版本：

| 指标 | 数值 |
|------|------|
| 参数量 | 2.6M |
| 计算量 | 6.6 GFLOPs |
| 层数 | 181 |
| 检测头 | 3 个（P3/P4/P5，stride 8/16/32） |

**核心架构：**

```
输入图像
  │
  ├── Backbone（特征提取）
  │     Conv → Conv → C3k2 → Conv → C3k2 → Conv → C3k2 → Conv → C3k2 → SPPF → C2PSA
  │     P1/2   P2/4          P3/8          P4/16         P5/32
  │
  ├── Neck（特征融合，PANet 双向路径）
  │     上采样 + Concat → C3k2（自顶向下融合大尺度语义信息到小尺度）
  │     下采样 + Concat → C3k2（自底向上融合小尺度细节到大尺度）
  │
  └── Head（检测头）
        Detect(P3, P4, P5) → 分别负责小/中/大目标
```

**关键模块：**
- **C3k2**：CSP Bottleneck 的变体，使用两个卷积核交叉连接，提取多尺度特征
- **C2PSA**：Cross Stage Partial + Spatial Attention，在特征提取末端增强空间注意力
- **SPPF**：Spatial Pyramid Pooling - Fast，多尺度池化聚合全局上下文信息

**YOLOv11n 在本任务的局限**：默认最小检测 stride=8，对于无人机图像中仅 ~50×50 像素的极小目标，特征图分辨率不够。

---

### 1.2 改进详解

#### 改进 1：P2 小目标检测头

**问题**：默认 3 个检测头（stride 8/16/32）对小目标检出率不足。

**方案**：新增 stride=4 的 P2 检测头，利用更高分辨率特征图。

```
原始：Detect(P3, P4, P5)    → 3 个检测头
改进：Detect(P2, P3, P4, P5) → 4 个检测头
                 ↑
          stride=4，特征图为输入的 1/4
          1280 输入 → 320×320 的检测网格
```

**原理**：P2 层保留了更多空间细节信息。对于 1280×1280 输入，P2 层生成 320×320 的特征图（比 P3 的 160×160 大 4 倍），每个网格单元覆盖原图仅 4×4 像素区域，大幅提升小目标的定位精度和检出率。

**论文来源**：TPH-YOLOv5（ICCV Workshop 2021）首次在 YOLO 中使用高分辨率检测头用于无人机场景。

---

#### 改进 2：SimAM 零参数注意力

**问题**：小目标容易淹没在复杂背景中，需要增强特征判别性，但 391 张小数据集不能承受额外参数。

**方案**：在 backbone 的 P3 和 P4 层后插入 SimAM（Simple Attention Module）。

```
原始：C3k2 → Conv → ...
改进：C3k2 → SimAM → Conv → ...
```

**原理**：SimAM 基于神经科学中的能量理论，定义每个神经元的重要性为抑制周围神经元所需的能量。推导出封闭解：

```
e* = (x - μ)² / (4(Σ(x-μ)²/n + λ)) + 0.5
attention = sigmoid(-e*)
```

**关键优势**：**零参数**，不增加任何可学习参数，不会在小数据集上过拟合。消融实验：+P2+SimAM 比 +P2 高约 0.01 mAP50。

**论文来源**：Yang et al., SimAM, ICML 2021

> **注意**：旧方案使用 EMA 注意力，但 EMA 引入额外参数导致早停（epoch 43），已全面替换为 SimAM。

---

#### 改进 3：PConv 轻量化卷积

**问题**：P2 检测头引入额外计算量，需要补偿以保持轻量化。

**方案**：在 Neck 中用 PConv_C3k2 替换标准 C3k2。

```
标准卷积：对全部 C 个通道做 3×3 卷积 → FLOPs = k²·C_in·C_out·H·W
PConv：  只对 C/4 个通道做卷积，其余直接传递 → FLOPs 降低约 75%

┌──────────────────┐
│  输入: C 个通道   │
├─────┬────────────┤
│ C/4 │  3C/4      │
│  ↓  │   ↓        │
│ 3×3 │ Identity   │
│ Conv │ (直接传递) │
├─────┴────────────┤
│ Concat → BN → SiLU│
└──────────────────┘
```

**原理**：相邻通道的特征往往高度冗余。PConv 只处理 1/4 的通道就能提取足够的特征信息，其余通道保持不变，大幅降低计算量而几乎不影响精度。

**效果**：模型参数减少 15-25%，FPS 提升，精度基本持平。

**论文来源**：Chen et al., "Run, Don't Walk: Chasing Higher FLOPS for Faster Neural Networks", CVPR 2023

---

#### 改进 4：SA-NWD 损失函数与标签分配

**问题**：IoU loss 的期望平方梯度 `G_IoU(s) ∝ s⁻²`，小目标梯度信号消失。

**方案**：用 SA-NWD（Scale-Adaptive Normalized Wasserstein Distance）替代 IoU。

```
原始：loss = 1 - IoU(pred, gt)
改进：loss = α · (1 - SA-NWD) + (1-α) · CIoU    （hybrid，α=0.5）
```

**原理**：将 bbox 建模为 2D 高斯，用 Wasserstein 距离衡量相似度。
自适应常数 `C_adapt = C_base · (1 + k/√S̄)` 基于**标注噪声正则化**设计：
小目标标注不确定性大（1px 误差对 20px 目标影响远大于 200px），
大 C 防止 loss 过度惩罚位置噪声，起到尺度自适应正则化效果。

> **注意**：`C_adapt` 不是 Fisher 等变最优推导的结果（Fisher 分析会给出 C∝s，方向相反）。
> 设计选择通过 Exp D（固定 C 对比）和 Exp E（反向 C 对比）消融实验验证。

同时替换 TAL 中的 IoU（SA-NWD-TAL），TAL 使用 `γ/δ` 对齐指数（与混合损失的 `α` 区分）。

**关键参数**：`c_base=12.0, k=1.0, alpha=0.5, nwd_min=0.3`

**消融结果**：+SA-NWD 使 mAP50 从 0.9600（基线）提升至 0.9705（+1.05%）。

> **注意**：`nwd_min=0.3` 必须设置，之前有 bug 写死 0.01 导致 P 崩溃，已修复。

**论文来源**：Wang et al., NWD, ISPRS 2022 + 本文正则化设计

---

#### 改进 5：Fisher-Guided CIoU（已归档，负贡献）

> **已归档**：所有相关实验（mAP50 0.827~0.937）均低于基线（0.960），已移入 `archive/scripts/`。
> 以下保留作为负例记录，**请勿在新实验中使用 `patch_scale_aware_loss` 或 `patch_sa_nwd_fisher_loss`**。

**原方案**：在 CIoU 基础上加 Fisher 补偿权重 `w(s) = √(ref_area/s)`。

```python
loss_iou = ((1 - ciou) * weight * scale_w).sum() / target_scores_sum
# scale_w = sqrt(0.002 / gt_area).clamp(0.8, 1.3)
```

**失败原因**：SA-NWD 用大 C 放松小目标的 loss（正则化方向），Fisher-CIoU 用大权重加强小目标的 loss（补偿方向），两者方向完全相反。组合使用导致训练信号混乱，效果比单独使用任一方法都差。

### 1.3 改进的整体效果

| 组件 | mAP50 影响 | 参数量影响 | FPS 影响 |
|------|:--------:|:--------:|:------:|
| P2 检测头 | ↑↑↑ | 略增 | 略降 |
| SimAM 注意力 | ↑ | **零参数** | ≈ 不变 |
| PConv 轻量化 | ≈ 不变 | ↓↓ | ↑ |
| SA-NWD Loss+TAL | ↑↑ | 无 | ≈ 不变 |
| Fisher-CIoU | ↓ 负贡献（已归档） | 无 | ≈ 不变 |
| BiFPN | ~~↑~~ **负贡献** | 略增 | 略降 |

核心思路：**P2 大幅提升精度，SA-NWD 解决度量问题，SimAM+PConv 零代价增强，BiFPN 已移除。**

## 七、常见问题

**Q: 训练时 GPU 显存不够怎么办？**
A: 将 batch 从 8 降到 4，或将 imgsz 从 1280 降到 960。

**Q: 训练 loss 不收敛？**
A: 检查 data.yaml 的 path 是否正确。小数据集建议关闭 mixup（设为 0）。

**Q: 消融实验某组报错？**
A: 大概率是自定义模块注册问题，确保 `import register_modules` 在最前面。

**Q: 如何训练 SOTA 对比模型？**
A: 用标准 ultralytics 命令即可：
```bash
yolo detect train model=yolov8n.pt  data=data.yaml imgsz=1280 epochs=300 batch=8
yolo detect train model=yolov10n.pt data=data.yaml imgsz=1280 epochs=300 batch=8
```
