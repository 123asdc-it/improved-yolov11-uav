## 第一章：Abstract（摘要）改稿提纲

### 1. 必须保留
- 任务场景：UAV 小目标检测。
- 核心问题：小尺度下 IoU 系列监督信号不足。
- 方法主线：SA-NWD + 轻量结构增强。
- 结果主线：精度提升 + 参数/速度代价。

### 2. 必须删除或替换
- 未完成实验或“进行中”描述。
- 与正文不一致的数据集/模型设定。
- 不可验证的强结论（如“strictly optimal”但无完整证据）。

### 3. 摘要模板（建议）
- 句 1：任务和痛点。
- 句 2：核心方法（只写最终版本）。
- 句 3：结构增强（P2/SimAM/PConv）。
- 句 4：关键结果（mean±std，主指标 + 效率）。
- 句 5：贡献价值（稳定性/泛化/部署）。

## 第二章：Introduction（引言）改稿提纲

### 1. 结构建议（5段）
- 段 1：应用价值与挑战（UAV 小目标）。
- 段 2：现有方法不足（架构改动多，度量问题少）。
- 段 3：你的核心洞察（尺度相关监督失衡）。
- 段 4：方法总览（核心方法层 + 架构增强层）。
- 段 5：贡献点（3~4 条，每条可被实验支撑）。

### 2. 贡献点改写规则
- 每条贡献必须映射一张表/一幅图。
- 不写“首次”除非可严格证实。
- 不把工程模块堆叠写成理论创新。

### 3. 引言里要新增的句子
- “我们将方法贡献与工程增强分开评估，以避免模块耦合造成归因不清。”
- “所有主结果基于多随机种子统计，并遵循统一训练预算。”

### 1. 章节顺序（建议）

## 第五章：Experiments（实验）改稿提纲

### 1. 先写“公平对比协议”小节
- 数据划分、输入尺寸、训练 epoch、增强策略、优化器、硬件。
- 对所有方法尽量同预算；若不同必须说明原因。

### 2. 主结果表（必须升级）
- 每个方法报告：mAP50、mAP50:95、Precision、Recall、F1。
- 同时报告：Params、FLOPs、FPS、Latency、ModelSize。
- 全部关键行改为 mean±std（>=3 seeds）。

### 0. 数据集来源（必须补，审稿人必问）
- §IV Dataset 节需补充数据集出处引用（若来自学长论文加 `\cite{}`，若自采需说明采集方式和标注协议）。
- 当前论文只写了"UAV drone detection dataset containing 559 images"，无来源，必须修复。

### 3. 消融设计（当前结构）
- 主链：Baseline → +SA-NWD(k=1) → +P2 Head（已有数据）。
- 自适应 C 方向验证：Fixed NWD (Exp D) / Reverse C (Exp E)（待填入）。
- 架构负例：+SimAM / +PConv / +BiFPN（已有，负贡献）。
- 目的：把“方法贡献”（NWD 度量）和“结构贡献”（P2 头）分离，并验证 C 设计方向。

### 4. 新增分析图
- **k vs mAP50 折线图**（Figure，Analysis 小节）：x 轴 k∈{0,0.5,1.0,2.0,3.0}，k-sweep 数据出来后用 `generate_paper_figs.py` 生成。k-sweep 结果**不加入主消融表**，单独在 Analysis 小节展示。
- 梯度幅值 vs 目标尺度（IoU vs SA-NWD 对比曲线）。
- 训练稳定性曲线（已有 `paper/figs/training_curves.pdf`）。

### 5. 统计显著性建议
- 报告 mean±std + 相对提升（%）。
- 可补充 t-test 或 bootstrap 置信区间（加分项）。

### 6. 失败案例分析（必须）
- 误检类型：背景纹理干扰、遮挡、密集小目标。
- 漏检类型：极低对比度、边缘目标、过小目标。
- 每类给 1~2 张可视化和简要原因。
