# Claude 工作规范（全局）

适用范围：所有项目。每次新项目将此文件复制到项目根目录。

---

## 1. 项目管理规范

### 归档规则（禁止直接删除任何文件）

| 内容类型 | 归档位置 | 备注 |
|---------|---------|------|
| 失败/过时实验脚本 | `archive/scripts/` | |
| 失败/过时模块代码 | `archive/modules/` | |
| 废弃函数 | 原文件保留，加 DEPRECATED 注释块 | 见下方格式规范 |
| AI 会话文件（`session-*.md`） | `archive/sessions/` | 含决策上下文，不得删除 |
| 历史结果文件 | `archive/results/` | |
| 其他过时文档 | `archive/` 根目录 | |

### DEPRECATED 注释格式

函数废弃时在函数定义前插入：

```python
# DEPRECATED: <废弃原因，一句话>
# 实验结果：<mAP/loss 等关键数据，无实验则写 N/A>
# 废弃时间：YYYY-MM-DD
# 替代方案：<当前推荐的替代函数或方案>
```

### 目录分类标准

| 目录 | 放什么 |
|------|--------|
| `scripts/`（或项目等效目录） | 当前活跃或计划中的实验脚本 |
| 核心模块目录 | 被活跃脚本 import 的模块 |
| `archive/` | 不再使用但保留参考价值的一切内容 |

## 2. Git 版本管理规范

每次任务结束前必须执行（无论改动大小）：

```bash
git add -A
git commit -m "<类型>: <简洁描述>"
git push
```

### Commit 类型约定

| 类型 | 适用场景 |
|------|---------|
| `exp` | 新增或修改实验脚本/模型代码 |
| `paper` | 论文内容修改（`.tex` / `.bib`） |
| `docs` | 文档更新（README / CLAUDE.md 等） |
| `archive` | 归档废弃内容 |
| `fix` | bug 修复 |
| `refactor` | 代码重构（不改功能） |
| `init` | 新项目初始化 |

## 3. 上下文记录规范

### 触发条件

每完成 **5 次实质性操作**（有文件改动的请求）后触发。
纯确认类消息（"好"、"继续"、"可以"等）不计数。

### 记录位置

在当前项目的 `docs/CLAUDE.md` 末尾追加快照。

### 快照格式

```markdown
## 上下文快照（YYYY-MM-DD，第 N 轮）

- 当前进展：<本轮完成了什么>
- 最新决定：<本轮确认的重要决策>
- 服务器/实验状态：<队列、进行中的实验>
- 待完成：<下一步最重要的 1-3 件事>
```

---

## 4. README 维护规范

### 规则

- 每个目录必须有 `README.md`
- 新增文件时同步更新对应目录的 `README.md`
- 归档文件时同步更新 `archive/` 及对应子目录的 `README.md`
- 函数签名或用途变更时同步更新模块目录的 `README.md`

### README 必须包含的内容

- 该目录的用途（1-2 句）
- 每个文件的说明（一行，含状态：活跃 / 待完成 / 废弃）
- 使用方法或注意事项（如适用）

---

## 5. 操作计数器

```
当前计数：0 / 5
上次快照：无

每次有文件改动的请求完成后此计数 +1。
达到 5 后：在项目 docs/CLAUDE.md 末尾记录快照，计数归零。
```
